Pazle's speed is gonna increase.

Dynamic languages are always slower and so there runtimes could be optimised to increase speed by surrendering so features like us, Now we can only do operations on numbers only. This is to increase pazle's speed but functions to do the missing exists.

Still we research and test on ways to make the language faster, its written in very fast programming languages D and C so I can figure out a way. Never the less, I found out one that is gonna affect the parsing and evaluation steps. This could do alot even without turning the program to a JIT. So please keep connected and in contact for the great change that is just around the corner as you learn the language.

If things go well, we may bring back the operations to strings and other datatypes so that we can have a real dynamic programming language that some of us dream of.
